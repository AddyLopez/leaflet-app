Leaflet Tutorial

Background Information:

What Is Leaflet?

According to its home page, Leaflet was created "11 years ago by Volodymyr Agafonkin, a Ukrainian citizen" and is "the leading open-source JavaScript library for mobile-friendly interactive maps."

The Purpose of This Tutorial:

This web page provides a short step-by-step tutorial for developers to get started with Leaflet in order to build interactive map features that can be integrated with other data. Learn how to initialize a map, create a marker with a pop-up, add a layers control, and include a geocoder to search for different cities.

Each instruction is assigned a number for quick identification, while each bullet point provides explanatory details as a supplement to clear up any possible confusion. It looks like a lot of text, but the process is actually quite simple. Run through these steps just once and a new world will open up for you to explore!

Check out the resources after the tutorial to continue on this journey. Although the focus here is on Leaflet in vanilla JavaScript, there are additional resources at the bottom for developers who are interested in learning React Leaflet. Familiarity with the simple steps outlined below will make it a lot easier to delve into the documentation to experiment with building more complex map features.

Personal Motivations:

Once I learned how to use Leaflet as an experimental design feature for my React weather application, I wanted to spread that knowledge to my development community at SheCodes. I also felt deeply moved by the inspiring examples on the pop-up on Leaflet's home page, which attest to the powerful storytelling that can be done with code and maps. I hope that you will feel inspired to take this knowledge further and create some cool apps! Happy coding!

Let's get started!

How to Initialize the Map:

  1. Import Leaflet's CSS and JS to the head of your index.html file by copying and pasting the link and script tags from the documentation.
  2. Create an empty div and assign it an ID to target the map (use a descriptive name, such as id = "map" or id = "leaflet-map").
  3. Inside a style tag in the head, assign width and height to the map using the ID's selector. It is a requirement to assign a height to the map for Leaflet to work. (Tip: a rule set of {width: 100%; height: 100vh;} will render a full-screen map).
    • To select an ID in CSS, prepend the name of the ID with a pound sign (for an ID of "map", write #map).
    • For more involved styling, consider linking to a separate stylesheet in the head tag of your HTML file.
    • For more information on how to style the map for mobile devices, see Leaflet on Mobile.
  4. Add a script tag above the body's closing tag.
    • Consider using a separate JS file for more involved projects. (Simply add an src attribute to the script tag and include the name of the file where you will write your JS, such as index.js, there.)
    • (Alternatively, if using a separate JS file, for the script file to load more quickly, you could include the script tag with the src attribute in the head and include the async attribute.)
  5. Inside the script tag (or your JS file), declare a map variable and use the syntax from Leaflet's overview. Explanation of the syntax below:
    • Here is the sample given:
      var map = L.map('map').setView([51.505, -0.09], 13);
      • On the left side of the assignment operator, the variable named map is declared. (Note to new developers: var is the pre-ES6 convention for declaring variables. Substitution of var by either of the now-common keywords let or const is acceptable here.
      • On the right side of the assignment operator, the L signifies "Leaflet."
      • After the dot operator and the method called map, include the map's ID from step 2 as the argument in parentheses. Wrap the ID in quotation marks.
      • After the next dot operator, the purpose of the setView method is to determine the default coordinates and zoom level for the map. The method takes in its parentheses the coordinates [latitude, longitude] as the first argument and an integer designating the zoom level as a second argument. The first and second arguments are separated by a comma.
        • A note about the zoom level: decrement the integer to zoom out; increment the integer to zoom in. (To learn more, see the documentation on zoom levels.)
  6. Include the next line of code from Leaflet's overview to add the tile layer to the map.
    • The tile layer includes a URL and an attribution, so that credit is given where credit is due.
      • In the map at the top of this web page, each of the map display options in the top-right corner (excluding the markers) represents a different tile layer. The attribution appears with a link to Leaflet at the bottom or bottom-right corner of the map.
      • To learn more about tile layers, check out this blog post.
    • Here is the sample given:
      L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {attribution: '© OpenStreetMap contributors'}).addTo(map);
      • Note: Copy the attribution's code directly from the documentation, not from the sample given here. The browser's user agent rendered the symbol for copyright and the anchor link wrapping "OpenStreetMap," interpreting the code instead of faithfully transcribing it character by character.
  7. Assign this code to a variable (for example, const openStreetMap = L.tileLayer etc.)
  8. For improved legibility, call the addTo(map) method on this new variable instead of at the end of the line above. For example: openStreetMap.addTo(map);
  9. Congratulations! You have initialized a map! Check the browser--by now, the OpenStreetMap tile layer should be visible as a map. If the image is broken, review steps 1 through 6 and use the information in the bullet points after each step to debug your application.

How to Add a Marker with a Pop-Up:

To create a default marker:

  1. Explore the map at the top of this web page. In the center, there is a blue icon that looks like an upside-down teardrop with a white eyelet. That is the default marker. Click on the marker to see the pop-up message!
  2. Go to the Marker section of Leaflet's documentation.
  3. Copy this line of code from the "Usage example":
    L.marker([50.5, 30.5]).addTo(map);
    • Explanation of the code above: This line instructs Leaflet (that is, L) to generate a marker at the coordinates specified. The in-built marker method takes as its first argument a latitude and a longitude separated by a comma and wrapped in square brackets. Notice that the latitude and longitude are NOT separate arguments. The addTo(map) method adds the marker to the map at the location designated by the given coordinates.
  4. Paste that line into your JS code anywhere beneath where you initialized the map with its first tile layer.
  5. Modify the latitude and longitude in square brackets to reflect the coordinates of the location where you wish to place the marker.
  6. To keep track of any icon that you create in this way, declare a variable and assign it to the line of code that generates the marker. Then you can remove the addTo(map) method from the end and instead call that method on the variable. For example:
    const defaultMarker = L.marker([50.4501, 33.5234]);
    defaultMarker.addTo(map);
    • Assigning the marker to a variable will make the logic of your code more intelligible when you want to add a pop-up.

To add a pop-up on this default marker:

  1. Visit the Popup section of Leaflet's documentation.
  2. Under the "Usage example," copy this line of code:
    marker.bindPopup(popupContent).openPopup();
    • Explanation of the code above: The bindPopup method is called on the marker in order to bind the pop-up's message to the marker. A placeholder for the pop-up content is given in parentheses as the bindPopup method's argument. Then, the openPopup method is called on the pop-up that is bound to the marker; its purpose is to enable the opening of the pop-up message on the marker.
      • As it says in the documentation, "Use Map.openPopup to open popups while making sure that only one popup is open at one time (recommended for usability), or use Map.addLayer to open as many as you want".
  3. Declare a variable for the pop-up and paste the code that you copied to the right of assignment operator. Substitute your marker's variable (from step 6 of "To create a default marker" in this tutorial) for marker. Substitute your message for the placeholder popupContent. Wrap your message in quotation marks within the parentheses to make it a string. For example:
    const defaultMarkerPopup = defaultMarker.bindPopup("I am the default marker's popup!").openPopup();
  4. Finally, with your code formatted in this manner, you can eliminate the addTo(map) method called on the default marker in step 6 of "To create a default marker" in this tutorial. Instead, you can add the default marker and the pop-up together in one line of code to the map. So, on the next line of your coding editor, call the addTo(map) method on the variable you declared for storing the pop-up. For example:
    defaultMarkerPopup.addTo(map);
  5. Now there should be a default marker on the map at the coordinates you specified with your pop-up message when you click on the marker! Check your map and return to these steps if you need to troubleshoot any problems.

For a customized marker:

  1. Explore the map at the top of this web page. Slightly off-center, there is a red icon of a thermometer. That is a marker with a customized icon. Click on the thermometer icon to see the pop-up message!
  2. Before we create the new marker, let's store the information for our customized icon in a new variable. Here is an example of how to generate a customized icon in Leaflet:
    const customIcon = L.icon({iconUrl: "./images/thermometer-icon-marker.png", iconSize: [45, 45], });
    • Explanation of the code above: The keyword const declares a constant variable named customIcon. After the assignment operator, Leaflet (that is, L) is instructed to generate an icon by using the icon method. In parentheses, the argument of the icon method is an object in curly braces. The object has two properties: iconUrl and iconSize, separated by a comma. The value of the property iconUrl is the path to the image file to be used as the customized icon. It is wrapped in quotation marks. The value of the property iconSize is an array with two integer values separated by a comma. The first integer value represents the width in pixels of the icon, while the second integer value represents its height in pixels.
    • Note: The value of the property iconUrl can be either the path to the image file or the URL at which that image is hosted.
    • To see other optional properties you could use to further customize the icon's appearance, go to Leaflet's documentation on Markers with Custom Icons.
  3. Now let's create a new marker that will use the customized icon. Declare a new variable to store the new marker. As with the default marker, you will have to specify the coordinates at which you would like the marker to appear. After the coordinates, include the icon option as a second parameter, like this:
    const customMarker = L.marker([50.4501, 30.5234], { icon: customIcon, });
    • Explanation of the code above: The keyword const declares a new constant variable named customMarker. After the assignment operator, Leaflet (that is, L) is instructed to generate a marker. In this case, the marker method takes two arguments separated by a comma, with the two arguments wrapped in parentheses. The first argument is the latitude and longitude separated by a comma and wrapped in square brackets. The second argument uses curly braces and includes the icon option. The value of the property icon is the variable customIcon that we created in the previous step.
  4. To add the new marker with the customized icon to the map, call the addTo(map) method on the variable you created in the previous step. For example,
    customMarker.addTo(map);

To add a pop-up on this customized marker:

  1. This process is essentially the same as the process to add a pop-up on the default marker. The only difference is that the customized icon needed to be generated and included in a new marker before binding the pop-up to it. Below is an example of a pop-up on the customized marker that we created above:
    const customMarkerPopup = customMarker.bindPopup("I am a pop-up on the customized marker!").openPopup();
  2. Finally, when we add customMarkerPopup to the map using the addTo(map) method, we can refactor our code. Eliminate the line of code added in step 4 of "For a customized marker." When you add the following code, it will add the pop-up with the marker simultaneously to the map:
    customMarkerPopup.addTo(map);

To customize the pop-up display using CSS:

  • It is possible to change how the pop-up looks. Because markers are important for displaying data at particular locations, it is worth experimenting with DevTools in the browser to customize how data are being displayed. (By "DevTools," I mean the developer tools that are available when you "right-click" on a web page and select the "Inspect" option from the drop-down list.) By using DevTools to target elements on the map that reveal Leaflet's CSS classes, you can easily override Leaflet's default styling and reassign properties to customize elements like pop-ups. For instance, in the map at the top of this web page, the default styling of the pop-ups, which had a white background and black text, has been changed to a black background with the text in white. Far more elaborate changes are possible.
  • You are welcome to experiment with my map, hack the current CSS using DevTools, and discover which of Leaflet's classes are controlling the default appearance of elements. That's how I learned.
  • Once you have identified which classes control what, you can target those classes from your CSS outside of the browser in your text editor to make the changes persist. Otherwise, the changes that you make in DevTools alone will disappear upon refreshing the page. Good luck, and have fun!

How to Add More Tile Layers and a Layers Control:

To add new tile layers:

  1. Browse Leaflet's tile layer providers here.
    • You will see a documentation window affixed to the top-center and a scrollable sidebar to the right.
  2. Simply select the tile layer you would like to add to your map, look in the documentation window for the section “Plain JavaScript,” and copy the code you see there. It will resemble the code used to add the OpenStreetMap tile layer in step 6 of "How to Initialize the Map" in this tutorial.
    • When you select an option from the sidebar, the map that forms the web page's main background will change to provide a preview of the tile layer selected.
    • If the preview is not visible, it is possible that that option is restricted to a particular range of zoom levels. Try zooming in or out using your mouse or trackpad to see if the preview activates. Otherwise, you might need an API key to view or use the tile layer.
  3. Once you paste that code from the documentation into the JS portion of your code, use the dot operator and call the addTo(map) method on the variable storing the tile layer, just as you did in step 8 of "How to Initialize the Map" in this tutorial.
    • Note: Although the tile layers on that page are free, some of them require an API key. Take a closer look at the tile layer's URL or the extra, parameterized options in curly braces sometimes provided after the URL for an indication that an API key is needed.
      • You can find out more information, such as which tile layer providers require registration, if you click here and scroll down or click the link at the top of the tile layer providers page that reads "Leaflet-providers" to get to the same web page.
  4. Feel free to repeat these steps to add multiple tile layers! (If you check the display right now, it might look a little chaotic. Don't worry, read on.)

To keep multiple tile layers organized and separable in the display, let's add a layers control:

  1. Take a quick look at the example map at the top of this web page. The layers control is represented by the large icon in the top-right corner. Hover the cursor over that icon. When you do so, the layers control will open and you will be able to select a tile layer by name to display.
    • The options with circles which are listed above the line in the layers control represent the base layers.
    • The options with squares which are listed below the line represent overlays, such as markers.
  2. Navigate to the Layers Control section of Leaflet’s documentation by clicking this link.
  3. Since you have already initialized your map and added the base layers, you can scroll past the first block of example code and go directly to this example:
    var baseMaps = {"OpenStreetMap": osm, "Mapbox Streets": streets};
    var overlayMaps = {"Cities": cities};
    • Explanation of the first code block above: The keyword var declares the variable baseMaps. The variable is assigned to an object. The object's properties are strings that are named after the tile layers included as base layers ("OpenStreetMap" and "Mapbox Streets"). The value of each property is the variable under which that tile layer is stored (in the example code, osm is the variable storing the tile layer for "OpenStreetMap").
      • However the property is named is how it will appear named in the tile layer controller on the map.
      • The order in which each property is declared in the object will determine the order in which each tile layer appears in the controller.
    • Explanation of the second code block above: The keyword var declares the variable overlayMaps. The variable is assigned to an object. The property "Cities" is a string; its value is the variable cities. The string "Cities" will appear in the tile layer controller on the map. The user will be able to check or uncheck a box next to "Cities" in order to use cities as an overlay to the map or to remove cities from the display.
      • To visualize this feature better, imagine that instead of "Cities" the property were "Marker" and that the variable stored a particular marker icon on the map. Without affecting the base layer map, the user could easily activate or deactivate that marker by checking the box next to "Marker" in the tile layer controller.
  4. Once you understand the example code above, use the first code block as the template to organize your layers. Declare a variable for your base layers. Assign that variable to an object. In between curly braces, assign properties that are strings and that correspond to the name of each tile layer that you have already added to the map. Follow each property's name with a colon. After the colon, assign a value. The value should be the variable that stores each corresponding tile layer (that is, the variable created in step 7 under "How to Initialize the Map" in this tutorial). Be sure to include a comma after the value when listing multiple properties in the object.
  5. If you have an markers or other overlays that you would like to organize in the layers control, use the second code block as the template. Declare a variable for your base layers. Assign that variable to an object. In between curly braces, assign properties that are strings and that correspond to the name of each marker or overlay that you have already added to the map. Follow each property's name with a colon. After the colon, assign a value. The value should be the variable that stores each corresponding marker or overlay. Be sure to include a comma after the value when listing multiple properties in the object.
  6. Head back to the Layers Control page of Leaflet's documentation. Scroll down to this line of code:
    var layerControl = L.control.layers(baseMaps, overlayMaps).addTo(map);
    • Explanation of the code above: The variable layerControl is declared. To the right of the assignment operator, the in-built control method tells Leaflet (that is, L) to add the layers provided in parentheses to the map. The first argument of the layers method is baseMaps, and it takes the base layers' object. The second argument of the layers method is overlayMaps, and it takes the overlays' object.
  7. Underneath the base layers' object and the overlays' object that you created in steps 4 and 5 above, paste the code that creates and adds the layers control to the map. You can eliminate the declaration of the variable and the assignment operator. Include everything to the right of the equal sign. Modify the layers method's parameters so that the names reflect the names you assigned to your base layers' object and/or your overlays' object.
    • If you want to include only the base layers' object, you do not have to include the second argument.
    • If you want to include only the overlays' object, pass null as the first argument and include the overlays' object as the second argument.
  8. Great! Check the browser to see your layers control! Is the icon appearing in the top-right corner? When you hover the mouse over the icon, does the layers control open and display the list of base layers and overlays without errors? When you select each option in turn, does the display change? If something went wrong, review the steps and information above.

How to Include a Geocoder Plug-in to Search for Cities on the Map:

  1. To start, glance at the map at the top of this web page. Look for the small icon of a magnifying glass in the top-right corner. When you click on the icon, a search bar will unfold. Type in the name of a city and press Enter. A drop-down menu of valid locations will appear. When you select a location from the list, the map's view will change to that location, and you will see a marker with an open pop-up that reads the location's name! That is the feature that the geocoder plug-in enables.
  2. Now that you know what we're building, go to the GitHub repository for the Leaflet Control Geocoder plug-in.
  3. Scroll down to the documentation underneath the repository. Go to the "Usage" section and copy both the link and script tags you see there.
  4. Paste that code into the head of your index.html file. You have now imported the geocoder plug-in's CSS and JS to your project.
  5. Assuming that you have already initialized your map and added a tile layer (such as OpenStreetMap) to the map, you can now add the geocoder to the map. Simply add this line of code to your JS code underneath where you initialized the map and added the tile layer(s):
    L.Control.geocoder().addTo(map);
    • By now, the addTo(map) method should be familiar. To read about the L.Control method for adding map controls, go to this section of Leaflet's documentation.
  6. That's it! Take a look at your map in the browser to verify that the icon of the magnifying glass is there and that the search engine is working.
    • If you want to explore some advanced customization options, return to the section labeled "Customizing" on the Leaflet Control Geocoder plug-in's documentation.
