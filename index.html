Leaflet Tutorial

Background Information:

What Is Leaflet?

According to its home page, Leaflet was created "11 years ago by Volodymyr Agafonkin, a Ukrainian citizen" and is "the leading open-source JavaScript library for mobile-friendly interactive maps."

The Purpose of This Tutorial:

This web page provides a short step-by-step tutorial for developers to get started with Leaflet in order to build interactive map features that can be integrated with other data. Learn how to initialize a map, create a marker with a pop-up, add a layers control, and include a geocoder to search for different cities.

Each instruction is assigned a number for quick identification, while each bullet point provides explanatory details as a supplement to clear up any possible confusion. It looks like a lot of text, but the process is actually quite simple. Run through these steps just once and a new world will open up for you to explore!

Check out the resources after the tutorial to continue on this journey. Although the focus here is on Leaflet in vanilla JavaScript, there are additional resources at the bottom for developers who are interested in learning React Leaflet. Familiarity with the simple steps outlined below will make it a lot easier to delve into the documentation to experiment with building more complex map features.

Personal Motivations:

Once I learned how to use Leaflet as an experimental design feature for my React weather application, I wanted to spread that knowledge to my development community at SheCodes. I also felt deeply moved by the inspiring examples on the pop-up on Leaflet's home page, which attest to the powerful storytelling that can be done with code and maps. I'm sharing this tutorial as a small tribute to the Ukrainian people by supporting Ukranian web development. I hope that you will feel inspired to take this knowledge further and create some cool apps! Happy coding!

Let's get started!

How to Initialize the Map:

  1. Import Leaflet's CSS and JS to your index.html file by copying and pasting the link and script tags from the documentation.
  2. Create an empty div and assign it an ID to target the map. (Use a descriptive name, such as "map" or "leaflet-map.")
  3. Inside a style tag in the head, assign width and height to the map using the ID's selector. It is a requirement to assign a height to the map for Leaflet to work. (Tip: a rule set of {width: 100%; height: 100vh;} will render a full-screen map).
    • To select an ID in CSS, prepend the name of the ID with a pound sign (for an ID of "map", write #map).
    • For more involved styling, consider linking to a separate stylesheet in the head tag of your HTML file.
    • For more information on how to style the map for mobile devices, see Leaflet on Mobile.
  4. Add a script tag above the body's closing tag.
    • Consider using a separate JS file for more involved projects. (Simply add an src attribute to the script tag and include the name of the file where you will write your JS, such as index.js, there.)
    • (Alternatively, if using a separate JS file, for the script file to load more quickly, you could include the script tag with the src attribute in the head and include the async attribute.)
  5. Inside the script tag (or your JS file), declare a map variable and use the syntax from Leaflet's overview. Explanation of the syntax below:
    • Here is the sample given:
      var map = L.map('map').setView([51.505, -0.09], 13);
      • On the left side of the assignment operator, the variable named map is declared. (Note to new developers: var is the pre-ES6 convention for declaring variables. Substitution of var by either of the now-common keywords let or const is acceptable here.
      • On the right side of the assignment operator, the L signifies "Leaflet."
      • After the dot operator and the method called map, include the map's ID from step 2 as the argument in parentheses. Wrap the ID in quotation marks.
      • After the next dot operator, the purpose of the setView method is to determine the default coordinates and zoom level for the map. The method takes in its parentheses the coordinates [latitude, longitude] as the first argument and an integer designating the zoom level as a second argument. The first and second arguments are separated by a comma.
        • A note about the zoom level: decrement the integer to zoom out; increment the integer to zoom in. (To learn more, see the documentation on zoom levels.)
  6. Include the next line of code from Leaflet's overview to add the tile layer to the map.
    • The tile layer includes a URL and an attribution, so that credit is given where credit is due.
      • In the map at the top of this web page, each of the map display options in the top-right corner (excluding the markers) represents a different tile layer. The attribution appears with a link to Leaflet at the bottom or bottom-right corner of the map.
      • To learn more about tile layers, check out this blog post.
    • Here is the sample given:
      L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {attribution: '© OpenStreetMap contributors'}).addTo(map);
      • Note: Copy the attribution's code directly from the documentation, not from the sample given here. The browser's user agent rendered the symbol for copyright and the anchor link wrapping "OpenStreetMap," interpreting the code instead of faithfully transcribing it character by character.
  7. Assign this code to a variable (for example, const openStreetMap = L.tileLayer etc.)
  8. Call the addTo(map) method on this new variable instead of at the end of the line above (for example, openStreetMap.addTo(map); for improved legibility).
  9. Congratulations! You have initialized a map! Check the browser--by now, the OpenStreetMap tile layer should be visible as a map. If the image is broken, review steps 1 through 6 and use the information in the bullet points after each step to debug your application.

How to Add More Tile Layers and a Layers Control:

  • To add new tile layers:
    1. Browse Leaflet's tile layer providers here.
      • You will see a documentation window affixed to the top-center and a scrollable sidebar to the right.
    2. Simply select the tile layer you would like to add to your map, look in the documentation window for the section “Plain JavaScript,” and copy the code you see there. It will resemble the code used to add the OpenStreetMap tile layer in step 6 of "How to Initialize the Map" in this tutorial.
      • When you select an option from the sidebar, the map that forms the web page's main background will change to provide a preview of the tile layer selected.
      • If the preview is not visible, it is possible that that option is restricted to a particular range of zoom levels. Try zooming in or out using your mouse or trackpad to see if the preview activates. Otherwise, you might need an API key to view or use the tile layer.
    3. Once you paste that code from the documentation into the JS portion of your code, use the dot operator and call the addTo(map) method on the variable storing the tile layer, just as you did in step 6 of "How to Initialize the Map" in this tutorial.
      • Note: Although the tile layers on that page are free, some of them require an API key. Take a closer look at the tile layer's URL or the extra, parameterized options sometimes provided after the URL for an indication that an API key is needed.
        • You can find out more information, such as which tile layer providers require registration, if you click here and scroll down or click the link at the top of the tile layer providers page that reads "Leaflet-providers" to get to the same web page.
    4. Feel free to repeat these steps to add multiple tile layers!

  • To keep multiple tile layers organized and separable in the display, let's add a layers control:
    1. Take a quick look at the example map at the top of this web page. The layers control is represented by the large icon in the top-right corner. Hover the cursor over that icon. When you do so, the layers control will open and you will be able to select a tile layer by name to display.
      • The options with circles which are listed above the line in the layers control represent the base layers.
      • The options with squares which are listed below the line represent overlays, such as markers.
    2. Navigate to the Layers Control section of Leaflet’s documentation by clicking this link.
    3. Since you have already initialized your map and added the base layers, you can scroll past the first block of example code and go directly to this example:
      var baseMaps = {"OpenStreetMap": osm, "Mapbox Streets": streets};
      var overlayMaps = {"Cities": cities};
      • Explanation of the first code block above: The keyword var declares the variable baseMaps. The variable is assigned to an object. The object's properties are strings that are named after the tile layers included as base layers ("OpenStreetMap" and "Mapbox Streets"). The value of each property is the variable under which that tile layer is stored (in the example code, osm is the variable storing the tile layer for "OpenStreetMap").
        • However the property is named is how it will appear named in the tile layer controller on the map.
        • The order in which each property is declared in the object will determine the order in which each tile layer appears in the controller.
      • Explanation of the second code block above: The keyword var declares the variable overlayMaps. The variable is assigned to an object. The property "Cities" is a string; its value is the variable cities. The string "Cities" will appear in the tile layer controller on the map. The user will be able to check or uncheck a box next to "Cities" in order to use cities as an overlay to the map or to remove cities from the display.
        • To visualize this feature better, imagine that instead of "Cities" the property were "Marker" and that the variable stored a particular marker icon on the map. Without affecting the base layer map, the user could easily activate or deactivate that marker by checking the box next to "Marker" in the tile layer controller.
